# Портфолио графического дизайнера — техническое задание (версия 2)
 
Документ для самостоятельной разработки на HTML / CSS / JS.
Бэкенд не требуется — сайт статический.
 
> **Что изменилось во 2-й версии**
> - Убрана вкладка/кнопка «Все работы» везде и отдельная плоская страница со всеми работами.
> - Блок «Избранное / Несколько последних кейсов» на главной заменён на **«Мои направления»**.
> - Вместо тизеров отдельных работ на главной — **7 категорий-направлений** (панелей).
> - У каждой панели-категории в правом нижнем углу — **число работ** в этой категории.
> - Клик по категории открывает **сетку работ** этой категории (тот же вид сетки).
> - Порядок категорий — **по числу работ** (больше работ = выше), при равенстве — **по алфавиту**.
 
---
 
## 1. Идея и задача
 
Персональный сайт-портфолио графического дизайнера.
Сайт решает две задачи одновременно:
 
- **Работодатель** (арт-директор, HR) — оценивает диапазон работ и мышление за ними. Ему важны раскрытые кейсы «задача → идея → результат» и возможность скачать CV.
- **Клиент на фриланс** — оценивает результат и удобство связи. Ему важны сильные работы в контексте и быстрый контакт.
Ядро для обоих одно — сильные работы, сгруппированные по направлениям. Различие только в двух точках входа: кнопка «Написать» (клиент) и кнопка «Скачать CV» (работодатель).
 
Ключевой принцип: **сам сайт — это часть портфолио**. Для дизайнера вёрстка, типографика и отступы читаются как проба навыка, поэтому качество исполнения сайта критично.

## 2. Архитектура сайта (карта страниц)
 
Многостраничная структура с настоящими URL. Навигация теперь двухуровневая: **направление → работы направления → отдельная работа**.
 
```
/                          Главная — витрина + «Мои направления»
/category-{slug}.html      Страница направления — сетка работ этой категории
/work-{slug}.html          Страница отдельной работы/кейса (по шаблону)
/about.html                Обо мне
```
 
Отдельной плоской страницы «Все работы» больше нет — её роль выполняют 7 страниц-направлений.
Отдельная страница «Контакты» не нужна — контакты живут в футере на всех страницах.
 
### Верхнее меню (одинаковое на всех страницах)
 
Слева — имя/логотип (ссылка на главную). Справа — пункты: **Работы**, **Обо мне**, **Контакты** (якорь на футер).
Пункт **«Работы»** ведёт на блок «Мои направления» на главной (`index.html#directions`), а не на отдельную страницу со всеми работами.
Меню фиксированное или обычное — на выбор; на мобильном сворачивается в бургер.

## 3. Страницы: структура блоков
 
### 3.1. Главная (`/`)
 
Короткая витрина, задача — заинтересовать и увести в нужное направление.
 
1. **Первый экран (Hero).** Имя крупно, роль одной строкой, одна главная кнопка («Смотреть работы» → якорь на «Мои направления»). Сильно работает крупная типографика или одна лучшая работа на весь экран.
2. **Мои направления.** Заголовок блока — «Мои направления». Сетка из **7 панелей-категорий** (см. раздел 3.2). У каждой панели в правом нижнем углу — число работ. Кнопки «Все работы» здесь нет.
3. **Обо мне кратко.** 2–3 предложения + фото. Ссылка «Подробнее» → `/about.html`.
4. **Футер** (контакты).
### 3.2. Направления (категории) и их страницы
 
**Семь направлений.** Панель категории = название + короткое пояснение (что входит) + число работ в правом нижнем углу. Вся панель кликабельна → страница направления `category-{slug}.html`.
 
| Направление | slug | Что входит |
|---|---|---|
| Айдентика | `identity` | лого, мокапы |
| UI/UX-дизайн | `uiux` | интерфейсы |
| Рекламный дизайн | `ads` | плакаты, баннеры, афиши, листовки, акции, промо |
| SMM | `smm` | посты в социальных сетях |
| Полиграфия | `print` | буклеты, меню, визитки |
| Инфографика | `infographics` | Ozon, Wildberries, Яндекс.Маркет |
| Motion Design | `motion` | анимированные баннеры, анимация логотипов, заставки, короткие видео |
 
**Порядок панелей на главной.**
Сортировка — по числу работ в категории, по убыванию (чем больше работ, тем выше/раньше).
При равном числе работ — по алфавиту.
Итоговый порядок выставляется, когда известно фактическое число работ в каждой категории; в разметке порядок панелей задаётся вручную под эти числа.
 
**Страница направления (`/category-{slug}.html`).**
1. Заголовок направления + короткое пояснение.
2. **Сетка работ** этой категории: 2–3 колонки на десктопе, 1 на мобильном. Карточка = превью-мокап + название + (опц.) подзаголовок. Вся карточка кликабельна → страница работы `work-{slug}.html`.
3. Ссылка «← Назад к направлениям» (`index.html#directions`).
4. Футер.
> Для «лёгких» направлений (SMM, инфографика, motion) работа может открываться сразу как крупная галерея без развёрнутого кейса. Для айдентики / UI-UX — как полноценный кейс (раздел 3.3). Шаблон один, глубина наполнения зависит от материала.
 
### 3.3. Страница работы (`/work-{slug}.html`) — ядро сайта
 
Единый шаблон, наполняемый для каждой работы. Структура сверху вниз:
 
1. **Шапка кейса.** Название, одна строка сути (клиент / задача), крупное главное изображение.
2. **Контекст.** Клиент, задача, роль дизайнера. Первое, что ищет работодатель.
3. **Идея и подход.** Текст о логике решения: что пробовала, почему выбрала это направление. Блок, отличающий кейс от галереи. Можно чередовать текст с картинками процесса.
4. **Результат (галерея карточек).** Сетка изображений финального результата. Крупные изображения, чередование полноширинных и парных.
5. **Итог.** Короткое завершение + результат для клиента, если известен.
6. **Навигация.** Ссылки «← Назад к направлению» (на свою категорию) и «Следующая работа» — чтобы страница не была тупиком.
7. Футер.
### 3.4. Обо мне (`/about.html`)
 
Текст от первого лица, инструменты (Illustrator, Photoshop, InDesign, Figma, After Effects — под motion), опыт, живое фото. 3–4 предложения. Кнопка «Скачать CV» (PDF).
 
### 3.5. Футер (на всех страницах)
 
Почта и Telegram крупно и кликабельно, ссылка на Behance, кнопка «Скачать CV», копирайт.
В футерной навигации — «Работы» (→ `index.html#directions`) и «Обо мне». Пункта «Все работы» нет.

## 4. UX-методы и принципы
 
- **Правило 3 секунд.** С первого экрана за 3 секунды понятно, кто перед вами и что делает.
- **Навигация по направлениям.** Пользователь сначала выбирает тип работ (направление), затем углубляется в конкретную работу. Число работ на панели помогает выбрать, где смотреть.
- **Двусторонняя навигация.** Со страницы работы легко вернуться в её направление и перейти к следующей работе. Никаких тупиковых страниц.
- **Прогрессивное раскрытие.** Главная даёт направления → страница направления даёт обзор работ → страница работы раскрывает глубину.
- **Одна главная кнопка на экран.** Один явный призыв, остальное вторично.
- **Работы в контексте.** Логотип на белом фоне продаёт хуже, чем тот же логотип на мокапе. Всегда показывать применение.
- **Единый шаблон.** Панель категории, карточка работы и страница работы — по одному шаблону каждая. Собирается один раз, дальше наполняется.
- **Mobile-first проверка.** Значимая доля просмотров — с телефона. Проверять каждую страницу на узком экране.
- **Доступность.** Контраст не ниже WCAG AA, у всех изображений `alt`, навигация с клавиатуры, кликабельные зоны не меньше 44×44 px.

## 5. UI / визуальный язык
 
- **Много воздуха.** Крупные отступы, работы «дышат».
- **Ограниченная палитра.** Нейтральная база + один акцентный цвет. Основной цвет дают сами работы.
- **Типографика.** Одна–две гарнитуры с характером. Чёткая иерархия. Ширина абзаца ~60–75 символов.
- **Сетка.** Единая модульная сетка, одинаковые вертикальные ритмы.
- **Панель направления.** Читаемое название, короткое пояснение, число работ в правом нижнем углу (моноширинный шрифт хорошо читается как «счётчик»). Лёгкий hover.
- **Сдержанные анимации.** Плавное появление блоков при скролле, лёгкий hover на карточках.
- **Единый стиль изображений.** Все мокапы желательно в одной подаче, чтобы сетка выглядела цельно.

## 6. Техническая реализация
 
Сайт статический — бэкенд не нужен, Python не требуется. Хостинг — GitHub Pages / Netlify / Cloudflare Pages (бесплатно).
 
### Стек
 
- **HTML** — семантическая разметка (`header`, `nav`, `main`, `section`, `article`, `footer`).
- **CSS** — один файл `style.css`. CSS Grid (сетки панелей и карточек) и Flexbox (меню, футер). Переменные `:root` для цветов, шрифтов, отступов. Медиазапросы. Счётчик в правом нижнем углу панели удобно позиционировать `position:absolute; right; bottom` внутри `position:relative` панели.
- **JS** — минимум: бургер-меню, анимации появления при скролле (IntersectionObserver). Без фреймворков.
### Число работ в панели категории
 
Два варианта:
- **Проще:** число проставляется вручную в разметке панели. Надёжно, но при добавлении работы число правится руками.
- **Автоматически (без бэка):** данные работ хранятся в `projects.js`; небольшой скрипт считает количество работ по каждой категории и подставляет число в панель и сортирует панели по этому числу. Одна точка правки при добавлении работы.
Рекомендация для «завайбкодить»: **начать с ручного числа**, перейти на авто-подсчёт, когда работ станет много.
 
### Как организовать страницы (для старта — отдельные файлы)
 
Каждое направление — свой `category-{slug}.html`, каждая работа — свой `work-{slug}.html`, по единым шаблонам. Плюс: работает без JS, легко хостить, хорошо для SEO. Минус: правка шаблона требует обновления файлов. Когда работ станет много — можно перейти на один шаблон + данные из `projects.js` с чтением `?id=` / `?cat=` из URL.
 
### Структура файлов
 
```
/
├── index.html
├── about.html
├── category-{slug}.html    (7 штук: identity, uiux, ads, smm, print, infographics, motion)
├── work-{slug}.html        (по одному на работу)
├── css/
│   └── style.css
├── js/
│   └── main.js
├── assets/
│   ├── img/                (превью, мокапы, фото)
│   └── cv.pdf
└── favicon.ico
```
 
### Схема данных
 
**Категория:**
- `title` — название направления
- `slug` — часть URL (`identity`, `uiux`, `ads`, `smm`, `print`, `infographics`, `motion`)
- `blurb` — короткое пояснение (что входит)
- `count` — число работ (для панели и сортировки)
**Работа:**
- `title` — название
- `slug` — часть URL (латиница, через дефис)
- `category` — slug направления, к которому относится работа
- `summary` — одна строка сути
- `cover` — путь к главному изображению
- `client`, `task`, `role` — блок «Контекст»
- `idea` — текст «Идея и подход»
- `gallery[]` — массив изображений результата
- `outcome` — текст «Итог»

## 7. Порядок работы
 
1. Разложить готовые работы по 7 направлениям, посчитать количество в каждом.
2. Заполнить схему данных для каждой работы.
3. Свёрстать `index.html`: Hero + блок «Мои направления» (7 панелей со счётчиками). Определить CSS-переменные — это фундамент.
4. Отсортировать панели: по числу работ (убывание), при равенстве — по алфавиту.
5. Свёрстать шаблон страницы направления на одной категории, довести до идеала, размножить на 7.
6. Свёрстать шаблон страницы работы на одном кейсе, размножить на остальные.
7. Свёрстать `about.html`.
8. Проверить мобильную версию всех страниц.
9. Проверить доступность (контраст, `alt`, навигация с клавиатуры).
10. Опубликовать на статическом хостинге.
---
 
## 8. Что учесть
 
- Имя дизайнера должно стоять крупно на первом экране — это часть личного бренда.
- Число работ в правом нижнем углу панели — единый стиль на всех 7 панелях.
- Порядок панелей пересматривать при изменении числа работ в категориях.
- Каждая страница работы обязана иметь выход (назад к направлению / следующая работа).
  - Мобильная версия — вертикальная лента с крупными картинками, комфортная для листания.
